Release Version: 0.2.2 (beta)

Project Description

You Shout (ushout in short) is a service that lets you post comments to Youtube videos and view comments from everyone in playback-time.

Unlike traditional comments in a list, these playback-time comments will be displayed along with the video at exact timestamps when they were posted. So viewers can see what other viewers think about very specific part of the video.

The Technique

Since Youtube is not supporting this service yet, we save all the comments on our server. (So please do NOT post any credential information.) Comments are pre-loaded and updated constantly as long as the page is open. As the video plays, comments are shot onto the screen according to their timestamps.

We are not using a Shockwave(flash) player to display the comments (while the video player could be either Shockwave or HTML5, decided by Youtube). Instead, pure html5 code is used. Comments are moving in a transparent overlay, which means even Youtube Shockwave player supports fullscreen playback, that can not be utilized; the comment layer can not stay with the player in Shockwave fullscreen. An HTML5 approach can be used to make everything fullscreen (video player and the comment layer), however, it is not yet a stabilized feature of HTML5. We definitely will consider using HTML5 fullscreen (it's so much better) when it becomes a stable standard, but for now we are sticking with full-window mode. It is more convenient in a way when you want to customize the player size.

Known Issues

  • It is reported that, in some browsers & operating system combinations, comments posted by the viewer will not show up lively. [await further inspection]

  • In some browsers & operating system combinations, due to an issue with Youtube embedded player, video duration can not be fetched until the video starts loading. [this could be resolved by getting video duration from Youtube Data API, but potential issues may still rise with live videos, which do not have a duration]

  • It is possible that, when jumping between timeframes within the video, the comments displaying manager loads all the comments between that two timeframes, instead of just the ones at where the video is playing, and shoot them all out. [fixed in 0.2.1]

Change Log

May. 3rd 2013 Version 0.2.2. Fixed an issue with iframe updating. Previously when the iframe was loading the feeds, it would modify the browser history as if you keep visiting new pages. This issue was recently noticed and fixed by replacing the original Javascript that sets the location of the iframe with "location.replace". It should work in most modern browsers.
